# Lombok的学习

##### Lombok的简介

```java
	Lombok是一个JAVA库，可以自动插入编辑器并构建工具，为您的java增添色彩。
  永远不要再写另一个getter或equals方法，使用一个注释，您的类具有一个功能齐全的构建器，自动化您的日志记录变量等等。
```

##### Lombok的实现原理

```java
注解的两种解析方式：
  1.运行时解析
  2.编译时解析（通过插入式注解处理器实现）
注解的实现方式：
  1.java文件在经过javac进行编译的时候，会生成抽象语法树（AST）.
  2.生成后会经过插入式注解处理器进行处理，看注解有没有对语法树进行变动，如果有，进生成新的的抽象语法树，在进行插入式注解处理器处理直到语法树没有变动。
  3.然后根据语法树生成字节码，最后形成.class文件。
```

##### Lombok的常用注解

```java
常见方法类：
  1.@Getter 作用于属性和方法
  2.@Setter 作用于属性和方法
  3.@ToString 作用于方法
  4.@EqualsAndHashCode 作用于属性和方法
  5.@Data 包含以上所有注解的功能
  6.@Val  弱语言声明关键字，可以自动判定变量类型
  7.@NotNull 非空分校验，如果为空，抛出空指针异常
构造方法类：
	8.@NoArgsConstructor  无参构造方法
	9.@AllArgsConstructor 全参构造方法
	10.@RequiredArgsConstructor 必要的参数构造方法
其他类：
  11.@CleanUp 作用于需要关闭资源的变量，自动释放和关闭资源
  12.@Slf4j   日志注解，可以直接使用日志打印
  13.@Builder 简化对象创建过程
  14.@Singluar 配合@Builder使用，简化集合类型的创建过程
```

##### Lombok的优缺点

```java
优点：
	1.通过注解自动生成样板代码，提高开发效率
  2.代码简洁，只关注相关属性
  3.新增属性后，无序可以修改相关方法
缺点：
  1.降低了源代码的可读性和完整性
  2.加大对问题的排查难度
  3.需要在idea中安装相关的插件
```
